System Documentation

1.1 Background of the Project

Poultry farming is a critical sector of global agriculture, contributing significantly to food security and economic development. Traditional egg incubation methods often rely on manual monitoring, which is susceptible to environmental fluctuations and human error, leading to low hatchability rates and financial losses for small-scale farmers.

1.2 Problem Statement

  • Newly hatched chicks cannot move to regulate their temperature, risking overheating or chilling.
  • Traditional heating (stoves, charcoal) causes uneven temperature and releases harmful gases.
  • Exposure to dangerous gases increases disease risk for chicks and harms the environment.
  • Lack of automation makes it hard for farmers to monitor and optimize farm conditions.
  • No proper management system prevents farmers from tracking performance and making informed decisions.

1.3 Objectives of the Project

Main Objectives:

The objectives are to create an automated system for the management of poultry using IoT technology that provides an appropriate environment for chicks to grow in after they are hatched, reduces human intervention in the process, increases the survival rate of chicks, and encourages sustainable farming practices in poultry. Additionally, the system will introduce machine learning for predicting diseases that can affect chicks, especially those caused by harmful gases and poor environmental conditions.

1.4 Scope of the System

The scope of this system covers the automated monitoring of environmental parameters within a closed hatchery environment. It includes hardware integration for data acquisition, a cloud-based communication layer, and a user interface for administrators and farmers. The system is limited to a single-stage incubation process monitoring.

1.5 System Architecture

Technical Implementation Layers:

  • Perception Layer: Real-time sensors for temperature, humidity, air quality (e.g., DHT22, MQ-135), and chick weight.
  • Processing Layer: ESP32 microcontroller for edge data processing, local alarms, and automated control of heating/ventilation.
  • Application Layer: Web dashboard (React), backend API (Node.js/Express), and MySQL database for data storage, visualization, and machine learning-based disease prediction.
